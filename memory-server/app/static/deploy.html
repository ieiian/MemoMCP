1. 前置条件

MemoMCP 依赖 PostgreSQL + pgvector 扩展来存储记忆和向量数据。

组件要求说明
PostgreSQL≥ 15（推荐 17）需安装 pgvector 扩展
Python≥ 3.12本地运行需要；Docker 部署不需要
Docker≥ 24.0Docker 部署需要
Docker Composev2Compose 部署需要
Embedding APIOpenAI / Gemini / 兼容服务用于生成记忆向量，必须配置其一

提示：使用 Docker Compose 部署时，会自动拉取 pgvector/pgvector:pg17 镜像，无需手动安装数据库。

2. 本地运行

本地运行适合开发调试，需要自行准备 PostgreSQL 数据库。

2.1 克隆项目

bash
git clone https://github.com/ieiian/MemoMCP.git
cd MemoMCP

2.2 准备数据库

使用 Docker 快速启动 PostgreSQL + pgvector：

bash
docker run -d \
  --name memomcp-postgres \
  -e POSTGRES_USER=memomcp \
  -e POSTGRES_PASSWORD=memomcp \
  -e POSTGRES_DB=memomcp \
  -p 5432:5432 \
  -v memomcp_pgdata:/var/lib/postgresql/data \
  -v $(pwd)/memory-server/init.sql:/docker-entrypoint-initdb.d/init.sql:ro \
  pgvector/pgvector:pg17

注意：init.sql 仅在数据库首次初始化时执行。如果数据卷已存在旧数据，需要先删除卷或手动执行 SQL。

2.3 安装 Python 依赖

bash
cd memory-server
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt

2.4 配置环境变量

在项目根目录创建 .env 文件（本地运行请将 DATABASE_URL 改为 localhost）：

.env（本地运行）
# ===== 运行模式 =====
AI_MEMORY_MANAGER=false

# ===== LLM Provider（仅 AI_MEMORY_MANAGER=true 时需要）=====
LLM_PROVIDER=gemini
LLM_MODEL=gemini-3.1-flash-lite
GEMINI_API_KEY=your-gemini-api-key
OPENAI_API_KEY=
OPENAI_BASE_URL=

# ===== Embedding Provider =====
EMBEDDING_PROVIDER=gemini
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_DIMENSION=1536

# ===== 数据库（本地运行用 localhost）=====
DATABASE_URL=postgresql+asyncpg://memomcp:memomcp@localhost:5432/memomcp

# ===== MCP 传输 =====
MCP_TRANSPORT=stdio

# ===== REST API =====
REST_HOST=0.0.0.0
REST_PORT=8000

# ===== 日志 =====
LOG_LEVEL=INFO

# ===== 管理台 =====
ADMIN_PASSWORD=your-admin-password

2.5 启动 REST API 服务

bash
cd memory-server
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

或使用项目脚本：bash run_local.sh

启动后访问：管理台 /admin | API 文档 /docs | 健康检查 GET /api/v1/health

2.6 启动 MCP 服务（远程模式）

bash
cd memory-server
python -m app.main --mcp --transport http --port 9000

MCP 端点：http://localhost:9000/mcp

3. Docker run 部署

使用 docker run 手动启动各组件，适合需要精细控制的场景。

3.1 创建 Docker 网络

bash
docker network create memomcp-net

3.2 启动 PostgreSQL

bash
docker volume create memomcp_pgdata
docker run -d \
  --name memomcp-postgres --network memomcp-net \
  -e POSTGRES_USER=memomcp -e POSTGRES_PASSWORD=memomcp -e POSTGRES_DB=memomcp \
  -p 5432:5432 \
  -v memomcp_pgdata:/var/lib/postgresql/data \
  -v $(pwd)/memory-server/init.sql:/docker-entrypoint-initdb.d/init.sql:ro \
  --health-cmd="pg_isready -U memomcp -d memomcp" \
  --health-interval=10s --health-timeout=5s --health-retries=5 \
  --restart unless-stopped \
  pgvector/pgvector:pg17

3.3 拉取或构建镜像

拉取
docker pull ieiian/memomcp-server:latest
构建
cd memory-server && docker build -t ieiian/memomcp-server:latest . && cd ..

3.4 准备 .env 并启动 REST API

在项目根目录创建 .env（完整内容见第 5 节），然后启动 REST 服务：

bash
docker run -d \
  --name memomcp-server --network memomcp-net \
  --env-file .env \
  -e DATABASE_URL=postgresql+asyncpg://memomcp:memomcp@memomcp-postgres:5432/memomcp \
  -p 8000:8000 --restart unless-stopped \
  ieiian/memomcp-server:latest

3.5 启动 MCP HTTP 服务

bash
docker run -d \
  --name memomcp-mcp --network memomcp-net \
  --env-file .env \
  -e DATABASE_URL=postgresql+asyncpg://memomcp:memomcp@memomcp-postgres:5432/memomcp \
  -p 9000:9000 --restart unless-stopped \
  ieiian/memomcp-server:latest \
  python -m app.main --mcp --transport http --port 9000

提示：确保 PostgreSQL 健康后再启动 server 和 mcp。推荐使用 Docker Compose 自动管理依赖。

4. Docker Compose 部署

项目提供三种 Compose 配置文件，均可在 GitHub 仓库 根目录找到。以下提供完整可复制的 docker-compose.yml 与 .env 内容。

快速开始：克隆仓库 → 创建 .env 并填入 API Key → 复制下方对应 compose 文件 → 执行 docker compose up -d

bash — 克隆项目
git clone https://github.com/ieiian/MemoMCP.git
cd MemoMCP

docker-compose.yml — Bridge 容器直连（推荐）

容器间通过 Docker 内部网络用服务名 postgres 直连，最通用稳定。将以下内容保存为项目根目录的 docker-compose.yml（仓库已自带，可直接使用）。

docker-compose.yml
# MemoMCP Docker Compose — Bridge 容器直连（默认）
# 启动: docker compose pull && docker compose up -d
services:
  init-db:
    image: ${MEMOMCP_IMAGE:-ieiian/memomcp-server:latest}
    container_name: memomcp-init-db
    user: "0:0"
    volumes:
      - db-init:/init-scripts
    command: >
      sh -c "cp /app/init.sql /init-scripts/init.sql
      && chmod 644 /init-scripts/init.sql
      && echo 'init.sql copied to shared volume'"
    restart: "no"

  postgres:
    image: pgvector/pgvector:pg17
    container_name: memomcp-postgres
    environment:
      POSTGRES_USER: memomcp
      POSTGRES_PASSWORD: memomcp
      POSTGRES_DB: memomcp
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
      - db-init:/docker-entrypoint-initdb.d:ro
    depends_on:
      init-db:
        condition: service_completed_successfully
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U memomcp -d memomcp"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s
    restart: unless-stopped
    networks:
      - memomcp-net

  memory-server:
    image: ${MEMOMCP_IMAGE:-ieiian/memomcp-server:latest}
    container_name: memomcp-server
    env_file:
      - .env
    environment:
      DATABASE_URL: postgresql+asyncpg://memomcp:memomcp@postgres:5432/memomcp
    ports:
      - "8000:8000"
    depends_on:
      postgres:
        condition: service_healthy
    healthcheck:
      test: ["CMD-SHELL", "python -c \"import urllib.request; urllib.request.urlopen('http://localhost:8000/api/v1/health')\" || exit 1"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 15s
    restart: unless-stopped
    networks:
      - memomcp-net

  memomcp-mcp:
    image: ${MEMOMCP_IMAGE:-ieiian/memomcp-server:latest}
    container_name: memomcp-mcp
    env_file:
      - .env
    environment:
      DATABASE_URL: postgresql+asyncpg://memomcp:memomcp@postgres:5432/memomcp
    command: python -m app.main --mcp --transport http --port 9000
    ports:
      - "9000:9000"
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - memomcp-net

volumes:
  pgdata:
    name: memomcp_pgdata
    driver: local
  db-init:
    name: memomcp_db_init
    driver: local

networks:
  memomcp-net:
    driver: bridge

5. 环境变量配置

完整 .env 模板（与仓库 .env.example 一致），可直接复制后按需修改：

.env（完整模板）
# ============================================================
# MemoMCP 环境变量配置
# 复制为 .env 并按需修改
# ============================================================

# ===== 运行模式 =====
# false = Passive Mode（默认，零 LLM 调用）
# true  = AI Memory Manager Mode（启用自动治理层）
AI_MEMORY_MANAGER=false

# ===== LLM Provider（仅 AI_MEMORY_MANAGER=true 时需要）=====
# 可选: gemini | openai | compatible
LLM_PROVIDER=gemini
LLM_MODEL=gemini-3.1-flash-lite

# Gemini
GEMINI_API_KEY=your-gemini-api-key

# OpenAI 官方
OPENAI_API_KEY=

# OpenAI Compatible（OpenRouter / DeepSeek / Moonshot / MiniMax / Ollama / vLLM / LocalAI）
# compatible 模式必填，例如: https://openrouter.ai/api/v1
OPENAI_BASE_URL=

# ===== Embedding Provider =====
# 可选: gemini | openai | compatible
EMBEDDING_PROVIDER=gemini
EMBEDDING_MODEL=text-embedding-3-small
# 向量维度（必须与 init.sql 一致；改维度需重建数据库）
# OpenAI text-embedding-3-small = 1536
# OpenAI text-embedding-3-large = 3072
# Gemini embedding-001 / text-embedding-004 = 768
EMBEDDING_DIMENSION=1536

# ===== 数据库 =====
# Compose Bridge（默认）: postgres
# Compose 宿主机绕行: host.docker.internal  → docker-compose.host-gateway.yml
# Compose Host Network: 127.0.0.1           → docker-compose.host.yml
# 本地运行: localhost
# 注意: docker-compose.yml 的 environment 会覆盖此项
DATABASE_URL=postgresql+asyncpg://memomcp:memomcp@postgres:5432/memomcp

MCP_TRANSPORT=stdio

# ===== REST API =====
REST_HOST=0.0.0.0
REST_PORT=8000

# ===== 日志 =====
LOG_LEVEL=INFO

# ===== 管理台 =====
# 管理台访问密码（/admin 页面）

13. 常见问题

Q: 数据库连接失败？

检查 DATABASE_URL：本地用 localhost，Docker Bridge 用服务名 postgres，Host Network 用 127.0.0.1。

Q: MCP 客户端连接不上？

  • stdio 模式：确保 cwd 路径正确，Python 环境已安装依赖
  • HTTP 模式：确保 MCP 服务已启动（端口 9000），防火墙未拦截
  • 检查日志：docker compose logs -f memomcp-mcp

Q: 向量维度不匹配？

EMBEDDING_DIMENSION 必须与 init.sql 一致（默认 1536）。修改需删除数据库卷重新初始化：docker compose down -v && docker compose up -d

Q: 如何更新？

bash
git pull origin main
docker compose pull
docker compose up -d

也可关注 GitHub 仓库 获取最新版本与 Release。

Q: 如何备份数据？

bash
docker exec memomcp-postgres pg_dump -U memomcp memomcp > backup.sql
docker exec -i memomcp-postgres psql -U memomcp memomcp < backup.sql
