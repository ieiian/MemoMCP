## Phase 4：MCP Server（已完成）
- 安装 FastMCP 3.4.3（清华镜像加速，支持 run()/run_async()/@mcp.tool 装饰器）
- app/tools.py：7 个 MCP 工具（save/search/update/delete/get/list/clear）
  - @mcp.tool 装饰器 + async def，参数用基本类型（str/int/float/list）
  - 每个工具用 async with async_session_factory() 管理 Session
  - 返回 dict（.model_dump(mode="json")）
  - 辅助函数 _validate_memory_type / _validate_uuid
- app/main.py 重构：双模式启动
  - 默认 REST API 模式（uvicorn app.main:app）
  - --mcp 参数切换到 MCP 模式（mcp.run(transport="stdio")）
  - --transport http 支持 HTTP 传输
  - app 变量保留在模块级别（uvicorn 兼容）
- database.py 修复：NullPool 替代默认连接池
  - 原因：MCP HTTP 模式下 engine 连接池绑定了 init_db() 的事件循环
  - NullPool 每次创建新连接，不绑定特定事件循环
  - REST API 模式也兼容
- mcp-config.example.json：Cursor / Claude Code / VSCode MCP 配置模板
- 实测验证：
  - 7 工具注册 ✓ | 直接调用(save/search/list/clear) ✓ | update/get/delete ✓
  - FastMCP Client HTTP 连接 ✓ | 工具列表 ✓ | 远程调用 ✓
  - REST API NullPool 兼容 ✓

## Phase 5：Embedding Provider（已完成）
- embedding/base.py：EmbeddingProvider ABC（dimension/embed/embed_batch/health_check）+ EmbeddingError
- embedding/gemini.py：GeminiEmbeddingProvider（text-embedding-004, 768维, httpx 调用 generativelanguage API）
- embedding/openai.py：OpenAIEmbeddingProvider（text-embedding-3-small, 1536维, /v1/embeddings）
- embedding/compatible.py：OpenAICompatibleEmbeddingProvider（继承 OpenAI, base_url 可配置, 支持 Ollama/vLLM/OpenRouter 等）
- embedding/__init__.py：工厂函数 get_embedding_provider()，根据 .env 创建 Provider，无 API Key 返回 None
- service.py 集成：
  - create_memory: 自动生成 embedding（title+content 拼接），失败不阻塞保存
  - update_memory: content 变更时重新生成 embedding
  - search_memories: Hybrid Search（向量+关键词 RRF 融合），无 Provider 回退关键词搜索
  - _rrf_merge: RRF (Reciprocal Rank Fusion) 融合两路搜索结果，k=60
  - _hybrid_search: 取 top_k*2 扩大召回，RRF 后截断
- 关键设计：优雅降级——无 API Key 时系统完全可用（关键词搜索），配置 API Key 后自动升级向量搜索
- 实测验证：
  - 无 API Key 回退 ✓ | 工厂创建 Gemini Provider ✓ | RRF 融合逻辑 ✓
  - REST API 端到端（创建+搜索）✓ | 无 embedding 时正常工作 ✓
