## 远程部署连接修复
- 问题：远程服务器 docker-compose 部署后 memomcp-server 报 ConnectionRefusedError [Errno 111]
- 根因：asyncpg 默认尝试 SSL 连接（_create_ssl_connection），PostgreSQL Docker 容器未配置 SSL
- 修复 database.py：engine 添加 connect_args={"ssl": False} 禁用 SSL
- 更新 .env.example：DATABASE_URL 添加注释说明 Docker 用 postgres / 本地用 localhost
- 用户需重新构建推送镜像：cd memory-server && docker build -t ieiian/memomcp-server:latest -f Dockerfile . && docker push ieiian/memomcp-server:latest
- 二次修复：SSL 已禁用但仍 ConnectionRefused，确认根因是 .env 中 DATABASE_URL 指向 localhost 覆盖了 docker-compose environment
- database.py 添加启动诊断日志：输出 DATABASE_URL 的 host:port，localhost 时输出 WARNING 提示用 postgres
- init.sql 目录问题：远程服务器上 init/init.sql 不存在导致 Docker 创建同名目录，PostgreSQL 初始化失败
- 最终修复：ImmortalWrt Docker bridge 网络隔离，容器间 TCP 连接被拒绝
  - docker-compose.yml DATABASE_URL 从 postgres:5432 改为 host.docker.internal:5432
  - 添加 extra_hosts: ["host.docker.internal:host-gateway"]
  - 通过宿主机端口绕过容器间网络隔离
  - 同时添加显式 networks: memomcp-net (bridge)
  - database.py init_db 添加 5 次重试 + DNS 诊断日志
